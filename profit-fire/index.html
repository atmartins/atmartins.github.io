Profit Fire is an interactive, site-specific video game/art piece.

Created by Aaron Martins and Sorob Louie.

Game Concept

Certain groups make a lot of money in unethical ways. War, pollution and exploiting labor are some common examples. Profit Fire attempts to satirize this behavior, and bring it to player's attention in a fun and engaging way.

Playing the Game

Due to the nature of this game, it will only be played during a one-night "showing" (located on the University of Oregon's campus). The primary purpose is to function as a piece of interactive art.

Players load the game on to their Android phones and connect to the server. Once the game is running, each player can choose from one of three "corporate sponsors" and begin firing projectiles at the target. The type of projectile that is fired is significant, and is determined by which "corporate sponsor" was chosen.

The phone acts as a window in to a 3D world, and to look around the player simply moves his or her phone in different directions. When the screen is tapped, a "fire!" signal is broadcast, and the server portion of the game displays a projectile hitting the actual part of the building. (See the video to get a better idea of how it works).

Technical Details

Profit Fire has two major components, the Android software and the Server software. The server software is based on the Unity game engine. Sorob Louie created the Unity part of the game and is responsible for network communication between the phone and the server.

The Android portion was written in Processing for Android by Aaron Martins. 2D artwork, 3D Models and texturing were created by Aaron as well.

Challenges

This project was created in under four weeks. All artwork, programming and 3D models, including the tank commander (60,000 poly model) were created during this process. Almost every line of code (650+ lines) for the Android game was written specifically for this game.

Calculating where a player touches the screen when the phone faces a certain direction was particularly challenging. The following formula was created for this game:
(tan(compass+domain adjustment)+x coordinate of screen being tapped*constant)*range scale adjustment.

Creating very lightweight 3D models, textures and images for use in the phone was challenging but fun. Most Android devices are fairly powerful, so there is a lot of room to grow with this type of thing.

Creating network communication with minimal lag was also a challenge, as well as getting the devices to communicate in the first place.
